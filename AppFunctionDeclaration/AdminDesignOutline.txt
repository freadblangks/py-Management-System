# 目前暂定按照以下数据结构设计了
  信息发送和服务器端管理界面逻辑

01  # 日志

	## 功能描述

		提供过滤器筛选查找信息


	# 过滤器类型

		当前任务状态
		任务所属单位

	# 一个任务的属性

		流水号  （每个任务的唯一编号）
		提交时间
		结束时间（若仍在处理，保持默认值无穷inf）
		任务类型（varchar类型 便于后期自行添加细分类型）
		提交用户（对应单位）
		任务状态
		文件路径（设置逻辑为：1.如果从客户端传输到服务器的文件路径存在，直接调用服务器端本身存在的数据 
				   2.否则接收客户端数据并将其保存到服务器本地，同时向数据库中写入保存的文件的路径）

	# 一个任务的状态
		
		已结束
		待处理（默认显示）

02  # 任务

	## 功能描述

		模块化开发，用于管理任务类型


	# 任务列表

		支持查看当前数据库中已经存在的任务

	# 添加任务

		支持添加特定类型任务
	
	# 修改任务
	
		支持删除、编辑特定类型任务

03 # 用户

	## 功能描述

		注册新用户
		管理用户的用户名和密码
	

04 # 导出
	
	## 功能描述
	
		支持一键导出excel表
	
05 # 帮助
	
	## 功能描述
	
		查看版本和帮助信息
